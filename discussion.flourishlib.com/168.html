Can't find some classes

posted by frank 10 years ago

Hello,

I have a problem with finding my classes on the production server. Under my develop machine it works great.

The difference between my develop and production machine is :

develop -> windows xp with xammp;

production -> unix apache

//For this example I replaced the domain name with <domain>!!

So the following standard flourish class seems to load just fine : /home/love/domains/<domain>/public_html/inc/flourish/fORMDatabase.php

But if i start using my own classes it just keeps popping up these Fatal error's : /home/love/domains/<domain>/public_html/inc/flourish/Dat_Member.php Fatal error: Class 'Dat_Member' not found in /home/love/domains/<domain>/public_html/login.php on line 106

Now the class i'm trying to autoload (Filename : Dat_member.php) :

class Dat_member extends fActiveRecord { protected function configure() { fORM::addCustomClassTableMapping($this, 'dat_member'); fGrammar::addCamelUnderscoreRule('Region1', 'region1'); fGrammar::addCamelUnderscoreRule('Region2', 'region2'); fGrammar::addCamelUnderscoreRule('Region3', 'region3'); fGrammar::addCamelUnderscoreRule('Region4', 'region4'); fGrammar::addCamelUnderscoreRule('Area1', 'area1'); fGrammar::addCamelUnderscoreRule('Area2', 'area2'); } }

The autoload class used : function autoload($class) { $flourish_file = DOC_ROOT . '/inc/flourish/' . $class . '.php'; if (file_exists($flourish_file)) { return require $flourish_file; } $file = DOC_ROOT . '/inc/classes/' . $class . '.php'; if (file_exists($file)) { return require $file; } throw new Exception('The class ' . $class . ' could not be loaded'); }

What am i doing wrong here on the production server?

With kind regards,

Frank van der Geld

Unix filesystems are case sensitive, whereas Windows ones are not and your class is called Dat_member, but your file is Dat_Member.php. Windows ignores the difference in case in the m, while Unix does not. You'll need to either rename the class to Dat_Member or the file to Dat_member.php.
