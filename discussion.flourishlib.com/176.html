fActiveRecord : don't have a constructor!

posted by julz_hk 10 years ago

Might help someone: 

$foo->populate, or $foo->setField('bar');

isn't working, make sure you do not have a standard __construct constructor in the class definition. But if you really must, then be sure to include this line in the method somewhere. 

parent::__construct();

Yes, is is essential that you let the fActiveRecord constructor execute.

If you would like to add some functionality, I recommend checking out the Adding Functionality to fActiveRecord section of the fORM documentation. You'll see there is a hook that will be executed after the normal constructor (post::__construct()), allowing you to add custom functionality without overriding it. This also has the benefit of being able to add the functionality to multiple fActiveRecord classes without having to override __construct() in each.
