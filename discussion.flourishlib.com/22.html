Multiple Databases

posted by appbeacon 10 years ago

I can't figure out how to deal with multiple databases that are running in the same instance of MySQL. I am connecting to the primary database with fDatabase and then attaching it. I can use fActiveRecord methods on tables in the primary database.

I CAN access the second database using normal SQL methods by prepending the db name to the front of the table name. Ex: Database = adhoc, table = tcell_itue_issues $result = $adhocdb->query('SELECT * FROM adhoc.tcell_itue_issues LIMIT 5');

However, when I don't see a way to use fActiveRecord to access the 2 different databases.

Things I've tried:

  • Using fDatabase twice to connect to each database and then attaching each to ORM
  • Using configure in my active record class to prepend the correct DB. Example:
class Tcell_itue_issue extends fActiveRecord
{
    protected function configure()
    {
		fORM::addCustomClassTableMapping($this, 'adhoc.tcell_itue_issues');
    }
}

None of these options work.

Any suggestions?

Unfortunately at this point the ORM is built strictly for connecting to a single database.

I would consider adding multi-database connectivity in the future, however I do a have a few large features in the works. If you would be interested in such functionality being added in the future, please file an enhancement ticket.
