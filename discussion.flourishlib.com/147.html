Flourish + Firebug + FirePHP

posted by rick 10 years ago

Currently, I have a quick and (very) dirty implementation of Firebug/FirePHP for exposing all of my Flourish debug information in Firebug's console instead of in the browser window.

The run down:

Added to: config.php 

include(DOC_ROOT . '/inc/flourish/FirePHP.class.php');

Patched in: fCore.php 

static public function expose($data)
{
  if ( function_exists('fb') ) {
    fb( htmlspecialchars((string) self::dump($data), ENT_QUOTES, 'UTF-8') );
  } 
  else {
    echo '<pre class="exposed">' . htmlspecialchars((string) self::dump($data), ENT_QUOTES, 'UTF-8') . '</pre>';
  }
}

Obviously this is complete garbage, but for now, in the development environment, it does what i need. Perhaps this is a candidate for inclusion to Flourish?

Welcome Rick! There has been some previous discussion about FirePHP, and from that came some fCore functionality that should make it fairly easy to integrate. You can find details at http://flourishlib.com/discussion/2/13.

For escaping output, if you have fHTML you can use fHTML::encode(): 

fb(fHTML::encode(fCore::dump($var)));
