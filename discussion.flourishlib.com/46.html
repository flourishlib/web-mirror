posted by mr. rossi 10 years ago

Hi, ORM validation seems different to mainstream frameworks. How should I code an e.g. "not empty" rule for a column represented in a model? Thanks, mr r.

The Flourish ORM derives the model from your database table. Because of this, any column that is set to NOT NULL will be required, otherwise the column will not be required. If you need to require a value and you can't set the column to NOT NULL, the simplest way is to override the schema using fSchema::setColumnInfoOverride(). Let me know if this is the case and I can post a code example using fORMSchema.
