You should check your code to see if the line breaks are being introduced by something you wrote. It is actually one of the Flourish code standards to not include closing PHP tags ?> to prevent exactly this issue. I checked the latest code base and confirmed that none exist.

In terms of encodings, yes, Flourish only works with UTF-8. You'll need to make sure you include the appropriate content-type headers to ensure that users see the correct output. The methods fHTML::sendHeader(), fXML::sendHeader() and fJSON::sendHeader() can help. The UTF-8 page has some more information about how to make sure all the data you deal with is encoded with UTF-8.

One of your source files has a byte order mark (BOM) for UTF-8 in it. See http://unicode.org/faq/utf_bom.html#bom5 for some more info about BOMs.

Be sure to check your editor to make sure it doesn't insert the BOM. I will looks to make sure I don't have a BOM in any of the Flourish files, however I'm fairly sure my editor does not use the BOM.
