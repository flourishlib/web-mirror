Defaults for POST/GET Suggestion

posted by appbeacon 10 years ago

This suggestion may be wasted on some, but it might help others.

When getting user input from $_POST or $_GET with fRequest::get(), you can use the default value to return false if the user did not provide a parameter.

My failure to do this was resulting in an fActiveRecord check against a null value. It could also cause different issues in your code.

Now, I do this: 


// Set default value to false in case user did not provide this parameter
$app_name = fRequest::get('app_name','string',false);
$cmd_groups_id = frequest::get('cmd_groups_id','integer',false);
$cmd = fRequest::get('cmd','string',false);
$replaceable = fRequest::get('replaceable','string',false);
$login = frequest::get('login','string',false);

// One simple if statement to ensure a value was provided for all required parameters.
if(!$app_name || !$cmd_groups_id || !$cmd || !$replaceable || !$login)
{
	$this->response = "Failed~~Not all parameters provided.";
	return;
}
