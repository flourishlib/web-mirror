fUpload on root directory

posted by antonio 10 years ago

Hi all!

I'm trying to use the fUpload class to upload an image on /my_products directory but seems that flourish ignore my destination directory request and uploads the file to the root .

This is the simple operation I'm trying to achieve the behavior I described above: 

$dir = new fDirectory(PRODUCTS_IMAGES);	
$file = $uploader->move($dir, 'image');

where PRODUCTS_IMAGES is set to /my_products The image is still uploaded on the root! Any ideas?

It is a little hard for me to tell what is happening here. Can you add the following code after what you've included? 

fCore::expose($dir);
fCore::expose($file);

The output of that should help me figure out what is going on.

posted by wbond 10 years ago

Thank you wbond for your answer, I figured out my problem. It was not responsible the *move* functiont but the *rename* (I was calling after the code pasted on the first post)

I was trying to rename the newly uploaded files giving as first argument the $path + $filename and as the second argument the $filename without the $path as I was convinced it was implicit for the function.

So I solved this problem which was only my misuse of the $uploader->rename() operation. Thnak you!
