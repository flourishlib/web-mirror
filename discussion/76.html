Creating Users for a site

posted by richard 10 years ago

has anyone created a tutorial on handling authentication for users and giving them certain permissions, I have read the documentation but not 100% certain on how to handle the construction of the users model.

I don't currently have a tutorial about users and authentication, and Im a little tight on time right now, so Im not sure when I will be able to get enhancing the fAuthorization documentation.

I think youve seen the Flourish Demo Site which uses the very basics to authenticate a single account. These basics are extended to specific auth levels and ACLs based on the project needs. The fAuthorization::require*() methods allow kicking the user to a login page when they don't meet the requirements for viewing a page.

Most of the sites I work on have an fActiveRecord class that works with a users table in the database. From there, I plug the user_id into fAuthorization::setUserToken() so that I can create the appropriate User object on each page load. I also use either the ACLs or auth levels to control page access. I set the auth level or ACLs during the login process once I've verified their login/password. You obviously don't need to use the ORM to get stuff out of the database, you can simply use fDatabase.

I know this doesn't include any code, but hopefully it can nudge you in the right direction. Perhaps someone else has some code they can share?
