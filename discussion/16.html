addCustomClassTableMapping problem

posted by vinegar 10 years ago

For some reason, fORM::addCustomClassTableMapping() is not working as expected. 

class Admin extends fActiveRecord {

    protected function configure(){
		fORM::addCustomClassTableMapping($this, 'this_admin_table');
	}

}

$check = new Admin();

produces: 

Warning: Illegal offset type in /mnt/fs/html/private/libs/flourish_r514/fORM.php on line 135

Fatal error: Uncaught exception 'fProgrammerException' with message 'The table specified, admins, does not exist in the database' in flourish_r514/fSchema.php:1407
Stack trace:
#0 flourish_r514/fActiveRecord.php(528): fSchema->getColumnInfo('admins')
#1 check.php: fActiveRecord->__construct()
#2 {main} thrown in flourish_r514/fSchema.php on line 1407

It just ignores the call and looks for the pluralised class name regardless.

I apologize, it appears the documentation was showing examples that weren't fully supported by the code.

As of r514 the first parameter needs to be the class name as a string, whereas the documentation was showing you could pass an object. If you download r517, the examples, and your code, will now work.
