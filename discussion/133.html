Guidance needed

posted by kris 10 years ago

Hi, I'm developing an application and looking for some advice.

The situation: Person A joins our training program he gives us the names of 10 of his friends who then fill out a survey for him. Person A completes the training then his 10 friends fill out the same survey and we figure out the improvement.

The particular problem I'm having is generating unique URLs for each survey. Using plain IDs like so is obviously undesirable:

www.survey.com/1-10-0

What I would like is an URL friendly hash like www.survey.com/NzcwNjgyNQ, that I can decode. I looked at fCryptography and Symmetric-Key Encryption but the hash generated is obviously not appropriate.

Any ideas?

Much thanks.

I would probably not worry so much about being able to decode the hash, and just add another column to your database table where you store a random 16 character code. If you set the column up as NOT NULL UNIQUE, you can ensure you will always have one 16 character code per survey.

The method fCryptography::randomString() will allow you to create a random string for each survey. The likelihood of generating the same random 16 character alphanumeric string is pretty close to zero.

If you are using the ORM you can load records by UNIQUE constrains by passing an associative array of the column_name => value to the constructor.
