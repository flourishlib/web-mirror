Disabling UTF-8

posted by appbeacon 10 years ago

I've decided to use flourish to rewrite a lot of the backend processing for AppBeacon. I love the default UTF-8 aspects of flourish for this project.

However, I have another project that is tied to a legacy system. It CANNOT use UTF-8. Unfortunately, for all the databases, we are stuck with latin1. Is there anyway to disable UTF-8 across the board in flourish? For example, I don't want it to automatically encode requests in UTF-8.

Suggestions?

Thanks, Justin Noel

Well, since you mention latin1, I presume you are using MySQL. When fDatabase first connects to the database, it sets the connection to use UTF-8 for all communication. This means that any communication to and from MySQL will use UTF-8.

The good news for you is that MySQL will automatically convert the UTF-8 communication to the encoding of your database. The only ill affects of this is that any characters that can not be stored by your database's encoding will become ?. If you are working with primarily English content, this shouldn't really be an issue.
