SERVER_NAME Suggestions

posted by appbeacon 10 years ago

When using flourish with CLI, flourish outputs this notice when Exceptions are found: 

Notice
-----
[internal function]: fCore::sendMessagesOnShutdown()
/home/ss7admin/jn_testing/gops_telcel/flourish/fCore.php(778): fCore::handleError(8, 'Undefined index...', '/home/ss7admin/...', 778, Array)
Undefined index:  SERVER_NAME

As you can see, fCore is not finding the index SERVER_NAME. Of course, this is because in the CLI environment, there is no SERVER_NAME.

I would suggest doing something to display the actual name of the server running the script. Perhaps something like this: 

if( isset($_SERVER['SERVER_NAME']) )
{
      $server = $_SERVER['SERVER_NAME'];
} else
      {
            $server = php_uname('n');
      }
}}

Yes, that does sound like a good idea. I'll open a ticket for this to be added to fCore. Thanks!
