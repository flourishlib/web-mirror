Do I need to run both setColumnCaseInsensitive and configureEmailColumn?

posted by marcus 10 years ago

Man I'm starting to feel like I'm pestering you with all these questions :P.

Anyway, do I need to run both fORMValidation::setColumnCaseInsensitive and fORMColumn::configureEmailColumn to make sure it's an email address and that noone enters the same email address but with other cases or does fORMColumn::configureEmailColumn do both?

Kindly, Marcus.

Yes, you would need to set both options. Be sure to note that fORMValidation::setColumnCaseInsensitive() only functions with columns in a UNIQUE or PRIMARY KEY constraint.
