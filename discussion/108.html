Can't add JS tags into template if file does not end in .js

posted by kris 10 years ago

Google's V3 maps API requires the developer to link to http://maps.google.com/maps/api/js?sensor=false which Flourish rejects owing to the fact the path does not end in .js

Please see the end of the section fTemplating#PlacingElements. There is a second optional parameter to force the type: 

// Force all data in the site_js element to be displayed in javascript tags
$tmpl->place('site_js', 'js');
