How can one store variables for below scenarios using flourish (!#3 especially)?

  1. Temp: Variables are accessible withing one script execution (like normal variables).
  2. Session: Variables that are accessible from all the pages for one user (saved in session).
  3. Application: Variables are accessible for all users from all pages (saved in file).

Well, the answers to these are going to depend on what scope you need the variables in.

  1. I would think these would be saved to the template or record, whichever makes more sense
  2. You can use fSession to store variables in a user-specific session that survives page reloads
  3. fCache allows for data to be save across all users, and possibly across all sites on the server depending on what backend you use. APC, XCache and Memcache will share across the server. File-backed caches will be shared across anything that uses the same file path.
